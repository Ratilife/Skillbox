Алгоритм записи данных из УФ 
При записи данных из формы, выполняются следующие этапы: 
// Первый раз, где мы можем что-то сделать и отказаться от записи. 
1.  На Клиенте. Форма: ПередЗаписью(Отказ, ПараметрыЗаписи) 
 
// Выполняется переход на Сервер. 
 
// Алгоритм обработка проверки заполнения: 
2.  ОбработкаПроверкиЗаполненияНаСервере(Отказ, ПроверяемыеРеквизиты) | Форма 
3.  ОбработкаПроверкиЗаполнения(Отказ, ПроверяемыеРеквизиты) | Модуль объекта 
4.  ОбработкаПроверкиЗаполнения(Источник, Отказ, ПроверяемыеРеквизиты) | Подписка 
 
// Заполняется Объект_СУБД и помещается в параметр ТекущийОбъект. 
// Дальнейшие изменения в реквизите формы «Объект» игнорируются! 
 
// Алгоритм перед записью. 
5.  ПередЗаписьюНаСервере(Отказ, ТекущийОбъект, ПараметрыЗаписи) | Форма 
6.  ПередЗаписью(Отказ, РежимЗаписи, РежимПроведения) | Модуль объекта 
7.  ПередЗаписью(Источник, Отказ, РежимЗаписи, РежимПроведения) | Подписка 
 
// Алгоритм при установке нового номера. 
8.  ПриУстановкеНовогоНомера(СтандартнаяОбработка, Префикс) | Модуль объекта 
9.  ПриУстановкеНовогоНомера(Источник, СтандартнаяОбработка, Префикс) | Подписка 
 
// Алгоритм записи объекта – открывается транзакция в СУБД. 
10. ПриЗаписи(Отказ) | Модуль объекта 
11. ПриЗаписи(Источник, Отказ) | Подписка 
 
12. ОбработкаПроведения(Отказ, РежимПроведения | Модуль объекта 
13. ОбработкаПроведения(Источник, Отказ, РежимПроведения | Подписка 
 
14. ПриЗаписиНаСервере(Отказ, ТекущийОбъект, ПараметрыЗаписи) | Форма 
 
// Запись завершена, отказ не возможен. 
// Объект_СУБД заново считывается в реквизит формы (реквизит формы полностью обновляется), 
// сам Объект_СУБД уничтожается. 
// Если мы заполняли какие-то дополнительные реквизиты объекта на форме (доп .колонки), 
// то они будут уничтожены. В последней процедуре следует их перезаполнить заново. 
 
15. ПослеЗаписиНаСервере(ТекущийОбъект, ПараметрыЗаписи) | Форма

// Возврат на Клиент. 
16. Форма клиент: ПослеЗаписи(ПараметрыЗаписи)
